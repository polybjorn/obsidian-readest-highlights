# Security policy

## Supported versions

Only the latest released version receives fixes. Older versions are not maintained.

## Reporting a vulnerability

Please use GitHub's [private vulnerability reporting](https://github.com/polybjorn/obsidian-readest-highlights/security/advisories/new) rather than opening a public issue. This keeps the disclosure private until a fix is available.

Expect an initial response within a week. If the report describes a real vulnerability, a fix and a coordinated disclosure timeline will be agreed before any public disclosure.
